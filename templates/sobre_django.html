MVC vs MTV en Django

Clásico (MVC) En Django (MTV) Rol
Model Model Representa los datos (base de datos)
View Template Presentación de la información (HTML)
Controller View Lógica de negocio y control del flujo

En Django, el controlador está implícito en el framework (urls.py + vistas).

Herencia de Plantillas

Template base (base.html)
<!-- base.html -->
<html>
  <body>
    {% include "partials/navbar.html" %}

    <main>
      {% block contenido %}
      {% endblock %}
    </main>

    {% include "partials/footer.html" %}
  </body>
</html>
Template hijo (home.html)
<!-- home.html -->
{% extends "base.html" %}

{% block titulo %}Inicio{% endblock %}

{% block contenido %}
  <h1>¡Bienvenido!</h1>
  <p>Contenido específico...</p>
{% endblock %}
Principio DRY aplicado: El header, navbar y footer se definen una sola vez en base.html y todas las páginas los heredan automáticamente usando {% extends %}.
{% endblock %}
